Multithreaded Web Caching Proxy Server


The specifications of this system are as follows

  1. Accept incoming requests for an XML file, and return the rendered HTML version from a cache or by forward the request to a target server
  2. Takes the following command-line arguments:
    1. Target server URL
    2. Port of the remote server serving XML that is not cached here
    3. Port on which this server will listen to requests
    4. Cache size in total number of files
Incoming requests to the server will be over HTTP and be a specific URL. For each incoming request:
  1. Check to see if an HTML (already converted from the source XML) version of the file is in your cache. If so, return that fileâ€™s contents.
  2. If not, forward the request to the target server
  3. When you get the response back (and it is valid), place the rendered HTML file in cache and then write the HTML contents back to the client.
  4. Each incoming request serviced in an individual thread
This implementation utilizes a raw socket connection to begin remote server transaction and all pertinent error codes are available.
