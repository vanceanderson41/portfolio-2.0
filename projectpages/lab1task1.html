Client to retrieve XML via HTTP request and transform it into HTML to be rendered


The specifications to this system are as follows

  1. Take command-line arguments for <server IP>, <port>, <name of file to be retrieved>,
  2. Handle errors where the XML file is no well-formed or invalid
  3. Transform the XML content to HTML
  4. Write the newly formed HTML to stdout
This particular implementation was done in Java using a raw socket connection based on the passed arguments and using using XSLT to manipulate the XML and retrieve HTML based on my template file
An example of how the available XML files will be formed:
<KnowledgeDocument>
	<KnowledgeAsset primarySubjectID="subject1">
	 	<title>XML Asset 1</title>
	  	<description>This is a sample KnowledgeAsset defined in the XML</description>
	   	<KnowledgeRating Rater="Joe" Rating="5" EntryDate="12-Dec-98">
	      	<description>Joe made a rating</description>
	  	</KnowledgeRating>
	  	<KnowledgeRating Rater="Sue" Rating="9" EntryDate="09-Mar-99">
	       	<description>Sue made a rating</description>
	  	</KnowledgeRating>
	</KnowledgeAsset>
	<KnowledgeSubject subjectID="subject1">
		<title>XML Subject 1</title>
	    <description>This is a sample KnowledgeSubject defined in the XML</description>
	</KnowledgeSubject>
</KnowledgeDocument>
